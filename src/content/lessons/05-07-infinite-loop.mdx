## 🎯 今天要做什么

学一个**超级重要的安全习惯**：循环里一定要留一句“等一等”，不然游戏会**卡死**！这一课没有新成果，但能帮你避开最常见的大坑。

## ✨ 新知识：没有 task.wait() 的循环会“卡死”

还记得第 3 课那个一直换色的方块吗？它能正常工作，是因为循环里有一句 `task.wait(1)`——每转一圈，**歇一秒**。

如果**漏掉**这句会怎样？

> 电脑会在**一秒里把循环转成千上万次**，忙到没空做别的事，整个游戏就**卡住不动**了，连 Stop 都很难点。

**对比一下：**

❌ 危险（不要这样）：

```lua
while true do
	print("hi")
end
```

✅ 好习惯（永远这样）：

```lua
while true do
	print("hi")
	task.wait(1)
end
```

两段只差一行 `task.wait(1)`，但一个会卡死、一个很顺畅。

<StudioStep n={1} title="回去检查你的循环">
  打开第 3 课那个让方块闪色的脚本，确认里面那句 **task.wait(1)** 还在。
</StudioStep>

<StudioStep n={2} title="给每个 while 都配上 task.wait()">
  以后每写一个 `while ... do`，都顺手在里面加一句 **task.wait()**（哪怕只是 `task.wait(0.1)`），让循环每圈都歇一下。
</StudioStep>

<StudioStep n={3} title="试玩，感受“顺畅”">
  点 ▶️ **Play**，玩一会儿。有了 task.wait()，方块照常闪色，游戏却很流畅、点哪都灵。
</StudioStep>

<Callout type="tip">
  记住一句口诀：**“while 循环里，必有 task.wait”**。养成这个习惯，你就基本不会写出卡死游戏的代码了。
</Callout>

## 🕹️ 看看你的成果

你现在拥有一个**老手才懂的好习惯**。以后游戏一卡，你第一个就会想到：“是不是哪个循环忘了 task.wait()？”——这正是会写代码的人解决问题的方式！⚠️

<Callout type="warning">
  万一真的卡住、Stop 点不动：等一两秒它常会自己缓过来；实在不行，关掉再打开 Studio。然后回去把漏掉的 task.wait() 补上。
</Callout>
